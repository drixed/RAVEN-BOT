# RAVEN BOT (external, screen-only)


- **захват экрана** (`mss`)
- **анализ изображения** (`opencv-python`)
- **управление мышью/клавиатурой** (`pydirectinput`)
- **привязку к окну игры** (`pywin32` / `win32gui`)

## Что умеет (весь функционал)

- **Выбор окна игры**: выбираешь окно из списка или берёшь активное.
- **Детект врага по тексту**: отслеживает появление/пропажу строки **«Нет Цель поиска»** в заданной области (Text ROI).
  - Используется шаблон `radar_empty_text.png` + `cv2.matchTemplate` с устойчивым препроцессингом.
  - Есть защита от “пустых/битых” кадров (счётчик текстовых пикселей) + гистерезис/подтверждение детекта.
- **Телепорт при враге**: при подтверждённом детекте нажимает `teleport_key`, ждёт `teleport_wait_s`, затем возвращается к фарму.
- **Пауза по хоткею**: можно поставить бота на паузу/продолжить глобальной клавишей (выбирается в UI).
- **Лог в файл (ночной режим)**: при включении сохраняет все логи в `logs\session_....log`.
- **Звук при враге**: при подтверждённом детекте может воспроизводить звук (beep или твой `.wav`).
- **Маршруты (Routes)**:
  - Шаги: **CLICK** (клик мышью), **KEY** (клавиша), **WAIT** (пауза).
  - CLICK умеет **ЛКМ и ПКМ** (button `left/right`).
  - Координаты кликов сохраняются **в процентах** (`x_pct/y_pct`) — меньше “съезжают” при изменении разрешения/масштаба окна.
- **Запись маршрута “вживую”**: запись кликов/клавиш через polling `GetAsyncKeyState` .
- **Конструктор маршрута по скриншоту**:
  - Делаешь скрин окна игры → кликаешь по картинке, добавляя точки/клавиши/паузы.
  - Точки можно **перетаскивать**.
  - Контекстное меню по шагам: удалить/дублировать/вставить WAIT/вставить KEY, редактировать delay/key.
- **Профили локаций**:
  - У каждого профиля свой набор маршрутов.
  - Для профиля выбираются 2 маршрута:
    - **Setup маршрут (вход в город/локацию)** — выполняется первым при старте.
    - **Farm маршрут (по карте)** — выполняется после setup и используется как “возврат к фарму” после ТП.
  - **Text ROI + `radar_empty_text.png` общие** для всех профилей (настройка детекта одна на всё).
- **AFK режим (без маршрута)**: можно включить фарм “стоим на месте”, без пробежки по маршруту.
- **Надёжные нажатия**: “удержание” клавиши `key_hold_s` .
- **Фокус окна**:
  - **Перехватывать фокус** (общий): может принудительно активировать игру для кликов/клавиш.
  - **Фокус только для ТП**: если выключен общий перехват, бот может принудительно сфокусировать игру **только когда надо телепортироваться**.

## Структура проекта

- `main.py` — запуск
- `ui.py` — интерфейс (Tkinter + ttkbootstrap)
- `bot_logic.py` — логика бота + маршруты + профили + сохранение настроек
- `recorder.py` — запись маршрутов (polling `GetAsyncKeyState`)
- `window_manager.py` — поиск окна, координаты, форс-фокус
- `input_controller.py` — клики/клавиши через `pydirectinput`
- `vision.py` — захват ROI и матчинги (в т.ч. по тексту)
- `points.json` — создаётся автоматически, хранит настройки/профили/маршруты
- `radar_empty_text.png` — шаблон текста “Нет Цель поиска” (должен существовать)

## Настройка с нуля (рекомендуемый порядок)

### Пошаговая инструкция (Setup/Farm) — чтобы было понятно когда Alt+Tab

1) **Выбери окно игры**
- Запусти игру.
- В UI нажми **«Взять активное»** (когда игра активна) или выбери окно из списка.
- Нажми **«Подтвердить»**.

2) **Создай профиль (локацию)**
- В **«Профиль (локация)»** нажми **«Новый профиль»** и назови его (например `city_1`).

3) **Setup маршрут (вход в город/локацию)** — записывается 1 раз
- Нажми **«Новый»** и задай имя (например `setup_city_1`).
- Нажми **«Старт запись»**.
- **Сразу Alt+Tab в игру** и сделай все действия руками (меню/подтверждения/ожидания загрузки).
- Когда закончил, **Alt+Tab обратно в бот** и нажми **«Стоп запись»**.
- Нажми **«Сохранить маршрут»**.
- В выпадающем **«Setup маршрут (вход в город)»** выбери `setup_...`.

4) **Farm маршрут (по карте/точкам)** — это “выход на фарм” и “возврат к фарму”
Вариант A (запись):
- **«Новый»** → имя (например `farm_city_1`)
- **«Старт запись»** → **Alt+Tab в игру** → сделай действия
- **Alt+Tab обратно** → **«Стоп запись»** → **«Сохранить маршрут»**

Вариант B (конструктор по скриншоту):
- Вкладка **«Маршрут»** → **«Снять скрин окна игры»**
- Инструменты **Точка/Клавиша/Пауза** → кликай по скрину, чтобы добавлять шаги
- **«Сохранить маршрут»**

5) **Выбери активный Farm маршрут**
- В выпадающем **«Farm маршрут (по карте)»** выбери `farm_...`.

6) **Запуск**
- **«Старт (маршрут)»**: выполнит Setup (если выбран) → потом Farm → включит детект.
- **«Старт AFK»**: **не запускает маршруты**, стоим на месте и включаем детект.

### 1) Выбери окно игры

1. Запусти игру.
2. В UI нажми **«Взять активное»** (когда игра активна) или выбери окно из списка.
3. Нажми **«Подтвердить»** (если есть).

Важно: все клики/координаты считаются **относительно окна игры**.

### 2) Настрой детект по тексту “Нет Цель поиска”

1. Открой вкладку **«Детект»**.
2. Настрой **Text ROI** — область, где появляется/исчезает текст **«Нет Цель поиска»**.
   - Удобнее всего: выделить ROI на скриншоте окна (если есть кнопка выбора ROI в UI).
3. Сделай шаблон `radar_empty_text.png`:
   - Встань в ситуацию “на радаре пусто” и текст **«Нет Цель поиска»** точно виден.
   - Сними скрин/шаблон (кнопка в UI делает файл `radar_empty_text.png`).
4. Нажми **Сохранить настройки** (если в UI есть кнопка).

Как работает детект:

- Пока текст **на месте**, match-score высокий → **врага нет**.
- Если текст **исчез** (score сильно ниже порога) несколько проверок подряд → **враг есть** → (если разрешено) ТП.

### 3) Создай профиль локации

Слева в блоке маршрутов:

1. В **«Профиль (локация)»** нажми **«Новый профиль»**.
2. Назови, например: `city_1` / `spot_2` и т.д.

### 4) Запиши Setup маршрут (вход в город/локацию)

Setup маршрут — это **точная запись действий**, чтобы попасть в нужный город/локацию:
клики по меню, подтверждения, ожидания загрузки.

Как записать:

1. Нажми **«Новый»**.
2. В поле **«Имя маршрута»** введи, например: `setup_city_1`.
3. Нажми **«Старт запись»**.
4. Переключись в игру и сделай действия руками:
   - открыть меню
   - выбрать город/локацию
   - подтвердить
   - при необходимости добавь ожидания (WAIT) — их можно вставить позже через контекстное меню.
5. Нажми **«Стоп запись»**.
6. Нажми **«Сохранить маршрут»**.
7. В выпадающем списке **«Setup маршрут (вход в город)»** выбери `setup_city_1`.

### 5) Запиши Farm маршрут (по карте)

Farm маршрут — это то, что ты делаешь **после входа**:
например, открыть карту → кликнуть точку → подождать → закрыть карту → и т.д.

1. Нажми **«Новый»**.
2. Имя маршрута, например: `farm_city_1`.
3. Запиши через **«Старт запись»** или собери в **конструкторе по скриншоту** (вкладка «Маршрут»).
4. **Сохрани маршрут**.
5. В выпадающем списке **«Farm маршрут (по карте)»** выбери `farm_city_1`.

### 6) Проверь настройки телепорта и фокуса

Во вкладке **«Настройки»**:

- **Клавиша телепорта** (`teleport_key`): обычно `f4`
- **Ожидание после телепорта** (`teleport_wait_s`): например 60 сек
- **Удержание клавиши** (`key_hold_s`): если игра “не видит” нажатия — увеличь (например 0.06 → 0.10)
- **ТП по врагу**: включает/выключает телепорт при детекте
- **Детект радара**: включает/выключает сам детект
- **Перехватывать фокус**: если включено — бот может активировать игру во время работы
- **Фокус только для ТП**: если включено — бот активирует игру только на момент телепорта (удобно, если ты часто Alt+Tab)

### 7) Старт

Нажми **Старт**:

1. Выполнится **Setup маршрут** (если выбран).
2. Затем выполнится **Farm маршрут** (если выбран и не включен AFK-режим).
3. Бот переходит в режим фарма/детекта.
4. При враге: подтверждает детект → делает ТП → ждёт → возвращается на фарм (Farm маршрут).

## Новые функции (быстрый гайд)

### Пауза/Продолжить (глобальный хоткей)
- Вкладка **«Настройки»** → **Хоткей → Пауза/Продолжить**: выбери кнопку из списка и нажми **«Применить хоткей»**.
- Эта клавиша работает **глобально** (даже если игра активна): нажал → бот встал на паузу, нажал ещё раз → продолжил.

### Лог в файл (ночной режим)
- Вкладка **«Настройки»** → **Лог в файл**:
  - включи **«Писать лог в файл»**
  - (опционально) поменяй папку (по умолчанию `logs`)
  - кнопки **«Открыть лог»** и **«Папка логов»** открывают текущий файл/папку
- Логи пишутся в отдельный файл **на каждую сессию**, например `logs\session_2026-04-23_23-10-05.log`.

### Звук при детекте врага (и свой WAV)
- Вкладка **«Настройки»** → **Сигнал при враге**:
  - включи **«Звук»**
  - выставь **«Раз»** (сколько повторов)
  - выставь **«Интервал (сек)»** (антиспам)
  - (опционально) в поле **WAV** выбери свой файл `.wav` (тогда будет проигрываться он, а не beep)

### TP_DEBUG (почему не ТП)
- Если ТП не подтвердился после 3 попыток, бот сохраняет кадры “до/после” в папку `tp_debug\` и пишет расширенный лог.

### Контрольная точка WAIT при записи
- В левом блоке записи маршрута есть **«Контрольная пауза (сек)»** и кнопка **«Вставить WAIT»**.
- Это добавляет шаг `WAIT` с заданной длительностью **прямо во время записи**, чтобы не ловить “время руками”.

### Меню/чат открыт → авто-ESC
- Вкладка **«Детект»** → блок **«Меню/чат открыт (авто-ESC)»**:
  - выбери ROI на индикаторе “меню/чат открыт”
  - сделай шаблон
  - включи тумблер
- Если меню/чат открыт, бот перед действиями нажмёт `ESC` и сделает клик по окну, чтобы вернуть управление игре.

### Смерть → маршрут восстановления
- Вкладка **«Детект»** → блок **«Смерть (детект + маршрут восстановления)»**:
  - выбери ROI на тексте/иконке смерти
  - сделай шаблон
  - выбери маршрут восстановления (воскреснуть/вернуться)
- При детекте смерти бот запускает этот маршрут (с cooldown, чтобы не зациклиться).

### Планировщик (старт/стоп по времени)
- Вкладка **«Настройки»** → блок **«Планировщик»**:
  - включи **Включить**
  - задай **Старт (HH:MM)** и **Длительность (ч)**
  - выбери режим `route` или `afk`
- Бот стартует **1 раз в день** и сам остановится по таймеру.

## Конструктор по скриншоту (вкладка «Маршрут»)

1. Нажми **«Снять скрин окна игры»**.
2. Выбери инструмент:
   - **Точка**: добавляет CLICK шаг по клику на картинке (ЛКМ = left, ПКМ = right).
   - **Клавиша**: добавляет KEY шаг (спросит клавишу).
   - **Пауза**: добавляет WAIT шаг (берёт delay из поля).
   - **Удалить**: удаляет точку со скрина.
3. Можно **перетаскивать** точки мышкой.
4. Список шагов справа:
   - ПКМ по шагу → контекстное меню (удалить/дублировать/вставить/редактировать).

## Описание вкладок UI (что означает каждая кнопка/переключатель)

### Вкладка «Настройки»

- **Сохранить**: сохраняет все параметры в `points.json`.
- **Ожидание после телепорта (сек)**: сколько ждать после ТП перед “возвратом к фарму”.
- **Удержание (сек)**: сколько удерживать клавиши (если игра пропускает нажатия — увеличь).
- **Delay шагов от/до (сек)**: общий рандом‑delay между шагами маршрута. Если **до=0**, delay фиксированный = **от**.
- **Клавиша телепорта**: клавиша для телепорта.
- **ТП по врагу**: включает/выключает телепорт при детекте.
- **Детект радара**: включает/выключает детект.
- **Вкл/Выкл**: быстро переключает детект без сохранения остальных настроек.
- **Фокус для ТП**: бот может фокусировать игру только на момент телепорта (удобно при Alt+Tab).
- **После ТП**:
  - `none`: ничего
  - `press_r`: нажать заданную клавишу
  - **Клавиша** и **Задержка** относятся к этому действию.
- **Авто‑подтверждать окна**: периодически жмёт заданную клавишу (например `y`) для подтверждений.
- **Перехватывать фокус игры**: если включено — бот может активировать игру для кликов/клавиш (в fullscreen иногда лучше выключить).
- **Фарм без маршрута (стоять на месте)**: AFK‑режим без маршрутов (стоим и только детект).
- **Старт (маршрут)**: Setup (если выбран) → Farm → детект.
- **Старт AFK**: без маршрутов, стоим + детект.

### Вкладка «Детект»

- **Text ROI (x, y, w, h)**: область внутри окна игры, где появляется текст **«Нет Цель поиска»**.
- **Выбрать Text ROI мышью**: откроет скрин окна, выдели область и нажми “Применить”.
- **Сделать скриншот 'Нет Цель поиска'**: сохранит шаблон `radar_empty_text.png`.
- **Порог совпадения**: насколько текущий текст должен совпадать с шаблоном.

Логика: текст **есть** → “пусто/нет цели”; текст **пропал** → “есть цель/враг” → (если включено) ТП.

#### Блок «Детект по урону + HP% (ТП по урону)»

- **Включить ТП по урону**: включает режим, где телепорт делается по индикатору “вас атакуют” + (опционально) по HP%.
- **ROI иконки атаки**: ROI именно на том месте, где **меняется** иконка (обычная → мечи).
- **Сделать шаблон обычной иконки**: нажимай когда тебя **не бьют** (обычное состояние).
- **Сделать шаблон иконки атаки**: нажимай когда тебя **бьют** (видны мечи).
- **Порог мечей / Порог обычной**: пороги совпадения (обычно 0.80–0.90).
- **Разница (margin)**: насколько `score(мечи)` должен быть выше `score(обычная)`, чтобы уверенно считать “атакуют” (обычно 0.03–0.06).
- **ТП только если HP ниже порога**: если включено — ТП спамится только когда HP% \(\le\) порога.
- **ROI HP**: ROI на красную HP‑полоску.
- **Порог HP (%)**: порог (например 70).
- **Жать ТП (раз) / Интервал / Cooldown**: сколько раз прожать ТП, интервал между нажатиями, и защита от спама.
- **Проверить HP сейчас / Проверить иконку атаки**: тесты, пишут в лог текущие `score` и HP%.

Примечание: при включенном “ТП по урону” обычное “ТП по врагу (радар)” **игнорируется**.

### Вкладка «Маршрут»

Конструктор по скриншоту:
- **Снять скрин окна игры**: делает скрин окна игры для конструктора.
- **Точка**: добавляет CLICK шаг по клику (ЛКМ=left, ПКМ=right).
- **Клавиша**: добавляет KEY шаг.
- **Пауза**: добавляет WAIT шаг.
- **Удалить**: удаляет точку/шаг по клику по маркеру.
- **Delay от/до (сек)**: задержка, которая запишется в шаги, добавленные в конструкторе (если **до=0**, delay фиксированный = **от**).

### Вкладка «Авто банки»

- **Включить автопокупку**: включает проверку количества банок (OCR).
- **Порог (<)**: если распознанное число меньше порога — бот выполняет маршрут покупки.
- **Ждать в городе (сек)**: пауза после маршрута покупки.
- **Работает при Старт (маршрут)**: автопокупка работает только при фарме по маршруту (не AFK).
- **Возврат по Farm после покупки**: после покупки выполнить Farm‑маршрут, чтобы вернуться на спот.
- **ROI банок (x,y,w,h)**: область в окне игры, где отображается число банок.
- **Выбрать ROI банок мышью**: откроет скрин окна игры, выдели область и нажми “Применить”.
- **Маршрут покупки**: маршрут, который ты заранее записал (домик → торговец → автозаполнение → покупка).
- **Проверить OCR сейчас**: покажет превью ROI и маску обработки, а также выведет debug в логи вкладки.

## Где хранятся данные

Файл `points.json`:

- **общие настройки**: окно игры, задержки, клавиши, Text ROI, пороги детекта, темы UI и т.д.
- **profiles**: список профилей (локаций), у каждого:
  - `routes` (маршруты профиля)
  - `setup_route_name` (какой маршрут запускать как setup)
  - `farm_route_name` (какой маршрут запускать как farm)

## Типичные проблемы и решения

- **Не нажимает клавиши в игре**:
  - увеличь `key_hold_s` (например 0.06 → 0.10/0.12)
  - включи **Перехватывать фокус** или **Фокус только для ТП**
- **Ложные телепорты**:
  - проверь, что Text ROI выделен точно по “Нет Цель поиска”
  - пересними `radar_empty_text.png` (чёткий текст, без полупрозрачных окон поверх)
  - увеличь `detect_confirm_streak` и/или порог `empty_text_threshold`
- **Маршрут “съезжает”**:
  - клики сохраняются в процентах, но всё равно важно, чтобы окно игры было примерно того же размера/масштаба
  - не меняй UI масштаб Windows/игры между записью и проигрыванием, если можешь
- **Alt+Tab мешает (бот возвращает фокус игре)**:
  - выключи общий **Перехватывать фокус**
  - оставь включённым только **Фокус только для ТП**
